üéØ Detailed Deployment Breakdown

1. Frontend Deployment (React App)

Where: Render Web Service

Service Type: Docker-based web service

Dockerfile: automation/frontend/Dockerfile

Build Process:

  1. Render pulls your code from GitHub/GitLab
  2. Builds Docker image using frontend/Dockerfile
  3. Runs npm run dev (Vite dev server)
  4. Exposes on HTTPS with automatic SSL certificate

What Users See

URL Path What It Shows Who Uses It
/ User Portal (PDF upload form) Citizens submitting address changes
/admin Admin Dashboard Supervisors reviewing cases

Environment Variables (Frontend)

VITE_API_URL=https://address-automation-backend.onrender.com

This tells the frontend where to send API requests

2. Backend Deployment (FastAPI + CrewAI)

Where: Render Web Service

Service Type: Docker-based web service

Dockerfile: automation/Dockerfile

Build Process:

  1. Render pulls your code from GitHub/GitLab
  2. Builds Docker image with Python dependencies
  3. Runs uvicorn src.automation.api:app --host 0.0.0.0 --port 8000
  4. Exposes on HTTPS with automatic SSL certificate

What Runs in Backend

  • FastAPI: REST API server
  • CrewAI: 7 sequential agents
  • MCP Tools: Database operations
  • OCR Service: Tesseract + GPT-4 parsing
  • Email Service: SendGrid integration
  • PDF Generation: reportlab for certificates

Environment Variables (Backend)

DATABASE_URL=postgresql://... (from Render PostgreSQL)
OPENAI_API_KEY=sk-proj-your-key
SENDGRID_API_KEY=SG.your-key
SENDER_EMAIL=no-reply@yourdomain.com

3. Database Deployment (PostgreSQL)

Where: Render PostgreSQL

Service Type: Managed PostgreSQL database

Version: PostgreSQL 16

Initialization: Runs docker/postgres/init.sql on first start

Database Tables

Table Purpose Key Columns
cases Store all case data case_id, citizen_name, status, canonical_address
audit_logs Track all operations case_id, timestamp, message

Database Connection

  • From Backend: Uses DATABASE_URL env variable (internal connection)
  • From Outside: Only accessible via Render internal network (secure)
  • Connection String: Provided automatically by Render

# Add GitHub remote (replace USERNAME/REPO)

# Push
git push -u origin main

3Create Render Account

  1. Go to https://render.com/register
  2. Sign up with GitHub account
  3. Authorize Render to access your repositories

4Deploy Database First

  1. Render Dashboard ‚Üí "New +" ‚Üí "PostgreSQL"
  2. Settings:
    • Name: address-automation-db
    • Database: address_db
    • Region: Choose closest to you
    • Plan: Free
  3. Click "Create Database"
  4. IMPORTANT: Copy the "Internal Database URL" (you'll need it for backend)

5Deploy Backend

  1. Render Dashboard ‚Üí "New +" ‚Üí "Web Service"
  2. Connect your GitHub repository
  3. Settings:
    • Name: address-automation-backend
    • Environment: Docker
    • Dockerfile Path: automation/Dockerfile
    • Region: Same as database
    • Plan: Free
  4. Add Environment Variables:
    • DATABASE_URL = (paste from step 4)
    • OPENAI_API_KEY = your OpenAI key
    • SENDGRID_API_KEY = your SendGrid key
    • SENDER_EMAIL = your verified sender email
  5. Click "Create Web Service"
  6. Wait 5-10 minutes for deployment
  7. COPY the service URL (e.g., https://address-automation-backend.onrender.com)

6Deploy Frontend

  1. Render Dashboard ‚Üí "New +" ‚Üí "Web Service"
  2. Connect same GitHub repository
  3. Settings:
    • Name: address-automation-frontend
    • Environment: Docker
    • Dockerfile Path: automation/frontend/Dockerfile
    • Region: Same as backend
    • Plan: Free
  4. Add Environment Variable:
    • VITE_API_URL = (paste backend URL from step 5)
  5. Click "Create Web Service"
  6. Wait 5-10 minutes for deployment
  7. THIS IS YOUR APP URL! (e.g., https://address-automation-frontend.onrender.com)

7Initialize Database Schema

The database needs the tables created. You have two options:

Option A: Using Render Shell (Recommended)

  1. Render Dashboard ‚Üí Backend Service ‚Üí "Shell" tab
  2. Run: 
    psql $DATABASE_URL -f docker/postgres/init.sql

Option B: Submit First Case (Auto-creates on error)

The application will create tables on first database error (less reliable)

Free Tier Limitations

  • Services Sleep: After 15 minutes of inactivity, free tier services sleep. First request takes 30-60 seconds to wake up.
  • Database Size: Free PostgreSQL has 1GB storage limit
  • Build Minutes: 500 free build minutes/month
  • No Custom Domain: Free tier uses .onrender.com subdomain

üö® Troubleshooting Common Issues

Issue: Frontend says "Network Error"

Solution: Check VITE_API_URL in frontend environment variables matches exact backend URL

Issue: Backend shows database errors

Solution: Verify DATABASE_URL and ensure database schema is initialized

Issue: Service keeps crashing

Solution: Check logs in Render dashboard, usually missing dependencies or env vars

Issue: Email not sending

Solution: Verify SendGrid API key and sender email is verified in SendGrid dashboard
